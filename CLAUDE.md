## What This Is

A Rails 8.1 API-only message gateway that consumes WhatsApp messages from RabbitMQ (via Evolution API webhooks), concatenates text messages within a sliding time window, transcribes audio via OpenAI Whisper, and publishes processed messages to another RabbitMQ queue. This is a background processing daemon, not a REST API -- the only HTTP endpoint is a health check at `/up`.

## Architecture

The message flow is: **RabbitMQ -> Consumer -> Solid Queue Job -> Strategy -> Publisher -> RabbitMQ**.

**Consumer layer** (`app/consumers/`): `ConsumerManager` starts one `MessagesConsumer` per queue listed in `RABBITMQ_QUEUES`. Consumers use manual ACK with prefetch(1). Retry count is tracked via `x-retry-count` message headers (not DLX) -- after `MESSAGE_MAX_RETRY_COUNT` failures, messages go to `<queue>.dlq` via `DeadLetterPublisher`.

**Job orchestration** (`app/jobs/`): `IncomingMessageJob` is the main coordinator -- it parses the payload into a `ParsedMessage` value object (Ruby `Data.define`), registers the sender, enqueues an async audit job, then dispatches to the appropriate strategy.

**Strategy pattern** (`app/services/strategies/`): `MessageStrategyResolver` routes by `message_type`:
- `"conversation"` -> `ConversationStrategy` -> `MessageConcatenationService`: appends text to a `ConcatenationBuffer` with a sliding window timer. Each message resets `expires_at`. A delayed `ConcatenationFlushJob` checks whether the timer truly expired (stale jobs are no-ops).
- `"audioMessage"` -> `AudioMessageStrategy` -> `AudioTranscriptionJob`: downloads audio, calls OpenAI Whisper, records `TokenUsage`, publishes result.

**Publisher layer** (`app/publishers/`): `ProcessedMessagePublisher` sends `{id, phone_number, text, name}` to `PROCESSED_MESSAGES_QUEUE`. `DeadLetterPublisher` sends failed messages with error context.

**RabbitMQ connection** (`lib/rabbit_mq/connection.rb`): Thread-safe singleton via Bunny with automatic recovery (10 attempts, 5s interval, 30s heartbeat).

## Key Conventions

- **UUID primary keys**: All models use string UUIDs generated by `ApplicationRecord#set_uuid` (before_create callback with `SecureRandom.uuid`).
- **Multi-database SQLite**: Separate databases for primary data, Solid Queue, Solid Cache, and Action Cable to reduce write contention. All use WAL mode.
- **RuboCop**: Inherits `rubocop-rails-omakase`. RSpec cops are relaxed -- `ExampleLength`, `MultipleExpectations`, `MessageSpies`, `AnyInstance` are all disabled; `NestedGroups` max is 4.
- **Test coverage**: SimpleCov enforces 80% minimum with branch coverage.
- **Test helpers**: `spec/support/` has `PayloadHelpers` with `build_text_message_payload` and `build_audio_message_payload` to construct Evolution API payloads. Tests use `FactoryBot`, `WebMock`, `ActiveJob::TestHelper`, and `TimeHelpers`.
- **Concurrency safety**: `ConcatenationBuffer` uses `with_lock` for safe concurrent appends.
